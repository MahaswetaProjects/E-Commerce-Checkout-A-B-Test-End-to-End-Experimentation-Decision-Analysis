## What Each Notebook Covers

### 01 — Data Quality & SRM Check
- Missing values, duplicate rows
- Contaminated users (assigned to both groups) — identified and removed
- Group/page mismatches (logging errors) — cleaned
- **Sample Ratio Mismatch check** via chi-square goodness-of-fit

### 02 — Frequentist Analysis
- Pre-analysis required sample size (were we powered enough?)
- Two-proportion Z-test
- **Wilson confidence intervals** (more accurate than normal approximation)
- **Cohen's h** effect size — is the difference practically meaningful?
- Post-hoc power analysis

### 03 — Bayesian Analysis
- Beta-Binomial conjugate model (no PyMC needed)
- 200,000 Monte Carlo samples
- **P(treatment > control)**
- **Expected loss** for ship vs no-ship decision
- 95% credible interval on the difference
- Monte Carlo convergence plot

### 04 — Novelty Effect & Segment Analysis
- Daily conversion rates with 3-day rolling average
- **Novelty effect check** — does the gap narrow over time?
- Segment tests: day of week, time of day
- **Multiple testing correction** — Bonferroni and Benjamini-Hochberg (FDR)

### 05 — Business Recommendation
- 3-scenario revenue model (conservative / expected / optimistic)
- 9-point ship / no-ship checklist
- Final written recommendation with rationale and next steps

## Interview Talking Points

- *"I always start with an SRM check — if the group split is off, all p-values are invalid."*
- *"I use Wilson CIs instead of the normal approximation because it's more accurate near the boundaries of the proportion range."*
- *"Alongside the Z-test I run a Bayesian analysis — P(treatment > control) = 19% is more intuitive to stakeholders than a p-value."*
- *"I check for novelty effects by plotting daily conversion rates. A gap that narrows is not a real improvement."*
- *"When running segment tests I apply Bonferroni for strict control and BH for exploratory work — they often give different answers, both worth reporting."*
- *"The correct conclusion here is DO NOT SHIP. Most experiments fail. Knowing how to correctly conclude that — and what to do next — matters as much as detecting a winner."*
